# Annotation Tool GUI

A Streamlit web app for row-by-row annotation of JSON files generated by the [python-perturbation-generator](https://github.com/senemogluc/python-perturbation-generator) pipeline.

## What it does

Pick a `.json` file from `data/raw/` (or upload one), then label each item as one of three categories:

| Button | Label | Meaning |
|--------|-------|---------|
| ✅ Preserve | `1` | The perturbation preserves the original meaning |
| ❌ Alter | `0` | The perturbation alters the meaning |
| ❓ Not Sure | `-1` | Unclear |

Each click auto-saves progress to `data/annotated/<filename>.json`, so reopening the same file later resumes where you left off. You can also download the annotated file at any time.

## Setup

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Expected file format

Input `.json` files must be shaped like:

```json
{
  "1": [ { "source_text": "...", "perturbed_text": "...", "question_type": "...", "perturbation_type": "...", "...": "..." } ],
  "2": [ ... ],
  "_summary": { "...": "..." }
}
```

Top-level keys are question-group ids, each mapping to a list of perturbation items. `_summary` (if present) is passed through unchanged. Only these per-item fields are shown in the UI:

| Field | Required | Description |
|-------|----------|--------------|
| `source_text` | ✓ | Original sentence (shown as "Original Text") |
| `perturbed_text` | ✓ | Modified sentence |
| `question_type` | — | Displayed as metadata |
| `perturbation_type` | — | Displayed as metadata |

All other fields on each item (e.g. `bertscore_f1`, `nli_label`, `question_id`) are preserved on save but not shown. An `annotation` field is added to each item once annotated.
